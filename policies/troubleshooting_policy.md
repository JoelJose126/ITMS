# Troubleshooting & Technical Support Policy

This policy governs how agents handle "Issue Addressal" (Feature 7) for payment, app, or login failures.

## 1. Diagnostic Protocol
*   **Step-Wise Investigation:** Agents must gather device/error info before searching the KB.
*   **One-Fix-at-a-Time:** Provide only one technical solution per interaction. Do not overwhelm the user with multiple complex steps.
*   **Verification:** Always ask, "Did that fix the issue?" after providing a solution.

## 2. Payment Failure Escalation
*   **Financial Integrity:** If a payment fix fails (e.g., card update link doesn't work), agents must NOT ask for the credit card number in chat. They must immediately escalate to the **Financial Audit Human Queue**.

## 3. App/Bug Reporting
*   **Bug Confirmation:** If the agent identifies a recurring error code (e.g., 404 on Signup), it must log the ticket as `SubType: System Bug` and notify the user that the technical team is investigating.
